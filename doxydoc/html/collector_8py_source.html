23 """
24 An universal buffer to queue incoming requests.
25 
26 Several tasks on servers can be heavy and processing them
27 can take a lot of time and block other tasks allocating
28 a part of resources.
29 
30 The goal of this solution is to queue incoming request
31 and store only its reference to avoid chaining a lot
32 of operations until the returning a result.
33 
34 - All the requests are queued into a LIFO register
35  and released from there when a service processing
36  the requests have availables resources.
37 - When a result is returned, its referenced session
38  fires a deferred with result as reason.
39 
40 We can define a timeout to limit the processing time of request.
41 """

110  def make(self):

234 class Collector(object):
235  """
236  Provides a smart buffer of request.
237 
238  Because some requests can be long and heavy, this provider
239  allows to put the requests into a queue where other side may
240  treat previous requests until last added.
241 
242  During making of each request, an identifier and a new deferred
243  is created. This deferred is assigned to incomming session
244  of request which is waiting with its unfired deffered.
245 
246  When queued request is processed and a result is returned,
247  waiting related deferred is fired with this result.
248  """
249 
250  queue = None
251  sessions = None

253  def __init__(self, sessions=Sessions()):
254  self.queue = []
255 
256  self.sessions = sessions

259  def queue_and_process(self, ip, request):
260  """
261  Inserts request into the queue.
262 
263  Each queued request returns a deferred which will be fired
264  when a queued element is processed.
265 
266  @param request
267  """
268  uid, deferred = self.sessions.make()
269  self.queue.append((uid, ip, request))
270 
271  return deferred

279  to get a request specification. This request is released from
280  internal queue, but its deferred still stored in session tracker.
281 
282  @return: uuid and request
